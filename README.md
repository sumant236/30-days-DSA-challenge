# 🏆 30 Days DSA Challenge (Java)

> 🚀 A rigorous 30-day coding marathon to master Data Structures and Algorithms. Focused on writing clean, optimized, and industry-standard Java code.

## 📖 About The Journey
This repository tracks my daily progress in mastering DSA. Instead of just solving random problems, I am following a structured path—starting from basic Recursion and Two Pointers, moving deep into Linked Lists, and currently conquering Stacks and Queues. 

Every implementation focuses heavily on **Time and Space Complexity Optimization**.

## 📂 Repository Structure & Progress

Here is the exact breakdown of my daily grind:

* **Day 1 & 2: Recursion** * Core recursive concepts, base cases, and call stack visualization.
* **Day 3 & 4: Two Pointers**
    * Optimizing $O(n^2)$ loops to $O(n)$ using dual-pointer setups.
* **Day 5: Sliding Window & Two Pointers**
    * Fixed and variable size windows for subarray/substring problems.
* **Day 6: Linked List Basics**
    * Node manipulation, traversals, and reversing.
* **Day 7: Linked List & Stacks (The Monotonic Series)**
    * Next Greater Element, Stock Span Problem, and the complex *Largest Rectangle in Histogram* using Monotonic Stacks.
* **Day 8: Queues & Cross-Implementations**
    * Queue Using Stacks, Stack Using Queues, Circular Queue.
* **Day 9: Revision Day1-8**
    * Sorting012, Min Stack, Max Sliding Window, Adding two Numbers in LinkedList

## 🔥 Key Technical Highlights
- **Amortized Analysis:** Implemented "Palti" (Two-Stack Flip) logic for efficient queue operations.
- **Memory Efficiency:** Built Circular Queues to prevent the $O(n)$ memory waste seen in standard array-based queues.
- **Monotonic Logic:** Mastered boundary-finding algorithms (Previous/Next Smaller Elements) for complex geometry problems like Histograms.
